## Overview

The Mythic Analytics JavaScript SDK captures events, page views, and user identity data from the browser. It manages batching, sessions, attribution, and remote configuration automatically.

Key capabilities:

- Initialize with a publishable key (`pk_` prefix, safe for client-side use)
- Track custom events with `capture` and page views with `pageview`
- Identify users with `identify` and create aliases with `alias`
- Manage super properties that attach to every event
- Control consent, opt-in/opt-out, and HIPAA compliance
- Auto-capture page views, page leaves, UTM parameters, and click IDs
- Remote configuration for tags, snippets, and transformers

<Callout kind="info">
  The SDK uses a publishable key starting with `pk_`. This key is safe to expose in client-side code. Never use admin or secret keys in the browser.
</Callout>

## Install or load the SDK

<Steps>
  <Step title="Add the CDN loader" icon="download" titleType="p">
    Add the CDN loader (~1.4KB) to your site. It sets up `window.mythic` immediately and asynchronously imports the SDK core:

    ```html
    <script src="https://api.adberserk.com/cdn/m.js"></script>
    ```
  </Step>

  <Step title="Initialize" icon="code" titleType="p">
    ```html
    <script src="https://api.adberserk.com/cdn/m.js"></script>
    <script>
      window.mythic.init("pk_your_publishable_key");

      // Methods can be called on window.mythic right away — calls made
      // before the core finishes loading are queued and replayed.
      window.mythic.capture("Signup started", { plan: "pro" });
    </script>
    ```
  </Step>
</Steps>

<Callout kind="info">
  With the CDN loader, the global is `window.mythic` (lowercase). It exposes pre-init stubs for every public method, so you can call `window.mythic.capture(...)`, `window.mythic.identify(...)`, etc. before initialization completes — those calls are queued and replayed once the core loads.
</Callout>

## Initialize the SDK

### `init(key, config?)`

Initialize Mythic. The CDN global exposes `window.mythic.init(...)`, which loads the SDK core, applies your config, and wires up the queued method stubs so calls made before init completes are replayed.

<ParamField body="key" param-type="string" required="true" show-location="false" deprecated="false">
  Publishable project key starting with `pk_`.
</ParamField>

<ParamField body="config" param-type="object" required="false" show-location="false" deprecated="false">
  Optional configuration overrides. See the [configuration reference](/javascript-sdk/configuration) for all options.
</ParamField>

```html
<script src="https://api.adberserk.com/cdn/m.js"></script>
<script>
  window.mythic.init("pk_your_publishable_key", {
    api_host: "https://analytics.yourdomain.com",
    debug: true,
    batch_size: 20,
    autocapture: true,
    capture_pageview: true,
    hipaa: false,
  });
</script>
```

<Callout kind="tip">
  When the SDK is loaded from a custom first-party domain (Cloudflare for SaaS), it auto-detects that hostname and uses it as `api_host` — you do not need to set `api_host` manually in that case.
</Callout>

## White-label the global object

By default the SDK attaches to `window.mythic`. You can rename that global — for example to your agency's or client's brand — so tracking calls read as `window.acme.capture(...)`. The chosen name must be a valid JavaScript identifier.

There are two ways to set it:

<Tabs>
  <Tab title="Per-client (recommended)" icon="settings">
    Set the client's `global_name` when you create or update it (see [Onboard a client](/for-agencies/onboard-a-client)). The loader snippet returned by `GET /api/v1/clients/{id}/loader` then bakes the override in automatically — you install the snippet as-is and the global is renamed for you.

    ```bash
    curl -X PATCH "https://mythic-analytics.gulp.workers.dev/api/v1/clients/acme-retail" \
      -H "Authorization: Bearer YOUR_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{ "global_name": "acme" }'
    ```
  </Tab>

  <Tab title="Inline on the page" icon="code">
    Set `window.__mythic_global_name` **before** the loader script runs. The loader reads it and attaches to that name (falling back to `mythic` if it isn't a valid identifier).

    ```html
    <script>window.__mythic_global_name = "acme";</script>
    <script src="https://api.adberserk.com/cdn/m.js"></script>
    <script>
      window.acme.init("pk_your_publishable_key");
      window.acme.capture("Signup started", { plan: "pro" });
    </script>
    ```
  </Tab>
</Tabs>

<Callout kind="info">
  `window.mythic` stays available as an alias even when a custom name is set, so shared snippets keep working. Storage keys also pick up the custom name by default (`{globalName}_{key}_`) — see [`persistence_name`](/javascript-sdk/configuration).
</Callout>
