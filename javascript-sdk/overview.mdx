## Overview

The Mythic Analytics JavaScript SDK captures events, page views, and user identity data from the browser. It manages batching, sessions, attribution, and remote configuration automatically.

Key capabilities:

- Initialize with a publishable key (`pk_` prefix, safe for client-side use)
- Track custom events with `capture` and page views with `pageview`
- Identify users with `identify` and create aliases with `alias`
- Manage super properties that attach to every event
- Control consent, opt-in/opt-out, and HIPAA compliance
- Auto-capture page views, page leaves, UTM parameters, and click IDs
- Remote configuration for tags, snippets, and transformers

<Callout kind="info">
  The SDK uses a publishable key starting with `pk_`. This key is safe to expose in client-side code. Never use admin or secret keys in the browser.
</Callout>

## Install or load the SDK

<Steps>
  <Step title="Load the script or install from npm" icon="download" titleType="p">
    For most sites, add the CDN loader (~1.4KB). It sets up `window.mythic` immediately and asynchronously imports the SDK core:

    ```html
    <script src="https://mythic-analytics.gulp.workers.dev/cdn/m.js"></script>
    ```

    Or install from npm for bundled apps:

    ```bash
    npm install @mythic-analytics/sdk
    # or
    yarn add @mythic-analytics/sdk
    ```
  </Step>

  <Step title="Initialize" icon="code" titleType="p">
    <CodeGroup tabs="Script tag,ES module">
      ```html
      <script src="https://mythic-analytics.gulp.workers.dev/cdn/m.js"></script>
      <script>
        window.mythic.init("pk_your_publishable_key");

        // Methods can be called on window.mythic right away — calls made
        // before the core finishes loading are queued and replayed.
        window.mythic.capture("Signup started", { plan: "pro" });
      </script>
      ```

      ```javascript
      import { init } from "@mythic-analytics/sdk";

      // init is async — it resolves once the SDK is initialized
      const mythic = await init("pk_your_publishable_key");
      mythic.capture("Signup started", { plan: "pro" });
      ```
    </CodeGroup>
  </Step>
</Steps>

<Callout kind="info">
  With the CDN loader, the global is `window.mythic` (lowercase). It exposes pre-init stubs for every public method, so you can call `window.mythic.capture(...)`, `window.mythic.identify(...)`, etc. before initialization completes — those calls are queued and replayed once the core loads.
</Callout>

## Initialize the SDK

### `init(key, config?)`

Initialize Mythic. The npm export is an async function that resolves to the SDK instance; the CDN global exposes `window.mythic.init(...)` which does the same and wires up the queued method stubs.

<ParamField body="key" param-type="string" required="true" show-location="false" deprecated="false">
  Publishable project key starting with `pk_`.
</ParamField>

<ParamField body="config" param-type="object" required="false" show-location="false" deprecated="false">
  Optional configuration overrides. See the [configuration reference](/javascript-sdk/configuration) for all options.
</ParamField>

```javascript
import { init } from "@mythic-analytics/sdk";

const mythic = await init("pk_your_publishable_key", {
  api_host: "https://analytics.yourdomain.com",
  debug: true,
  batch_size: 20,
  autocapture: true,
  capture_pageview: true,
  hipaa: false,
});
```

<Callout kind="tip">
  When the SDK is loaded from a custom first-party domain (Cloudflare for SaaS), it auto-detects that hostname and uses it as `api_host` — you do not need to set `api_host` manually in that case.
</Callout>
