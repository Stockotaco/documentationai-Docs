## Configuration reference

The full set of options you can pass to `init(key, config)`.

### Network and batching

<ParamField body="api_host" param-type="string" required="false" show-location="false" deprecated="false">
  Base URL for API calls. Default: `https://api.adberserk.com` (auto-detected from the host the loader script was served from). Change this if you use a custom domain or proxy.
</ParamField>

<ParamField body="batch_size" param-type="number" required="false" show-location="false" deprecated="false">
  Number of events to queue before sending. Default: `10`.
</ParamField>

<ParamField body="flush_interval" param-type="number" required="false" show-location="false" deprecated="false">
  Milliseconds between automatic flushes. Default: `2000`.
</ParamField>

<ParamField body="xhr_headers" param-type="object" required="false" show-location="false" deprecated="false">
  Custom headers to include with every network request.
</ParamField>

### Auto-capture

<ParamField body="autocapture" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Enable automatic capture of clicks, form submissions, and DOM interactions. Default: `true`. Pass an object to limit to specific element types: `{ elements: ['links', 'buttons', 'forms'] }`.
</ParamField>

<ParamField body="capture_pageview" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture a `$pageview` event on page load. Default: `true`.
</ParamField>

<ParamField body="capture_spa_pageview" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture page views on SPA navigation (History API pushState/replaceState). Default: `true`.
</ParamField>

<ParamField body="capture_pageleave" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture a `$pageleave` event with time-on-page and exit intent data. Default: `true`.
</ParamField>

<ParamField body="capture_utm" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture UTM parameters and advertising click IDs from the URL. Default: `true`.
</ParamField>

<ParamField body="capture_copied_text" param-type="boolean" required="false" show-location="false" deprecated="false">
  Track text copied to clipboard. Default: `false`.
</ParamField>

### Privacy and persistence

<ParamField body="persistence" param-type="string" required="false" show-location="false" deprecated="false">
  Storage strategy. Options: `'localStorage'`, `'cookie'`, `'memory'`, `'localStorage+cookie'`. Default: `'localStorage+cookie'`.
  When the browser blocks localStorage (private browsing, Firefox Focus), the
  localStorage modes degrade to tab-scoped `sessionStorage` — a multi-page visit
  keeps one device/session id (and one session replay) instead of minting fresh
  ids on every navigation, while nothing outlives the tab, honoring the
  browser's privacy stance. If sessionStorage is also blocked, `'localStorage'`
  falls to memory and `'localStorage+cookie'` to cookies. Explicit `'memory'`
  is never escalated.
</ParamField>

<ParamField body="persistence_name" param-type="string" required="false" show-location="false" deprecated="false">
  Custom prefix for storage keys. Defaults to `{globalName}_{key}_`.
</ParamField>

<ParamField body="cookie_name" param-type="string" required="false" show-location="false" deprecated="false">
  Custom cookie name when using cookie persistence.
</ParamField>

<ParamField body="cookie_expiration" param-type="number" required="false" show-location="false" deprecated="false">
  Cookie lifetime in days. Default: `365`.
</ParamField>

<ParamField body="cross_subdomain_cookie" param-type="boolean" required="false" show-location="false" deprecated="false">
  Share cookies across subdomains. Default: `true`.
</ParamField>

<ParamField body="secure_cookie" param-type="boolean" required="false" show-location="false" deprecated="false">
  Set the `Secure` flag on cookies. Default: `true` on HTTPS.
</ParamField>

<ParamField body="respect_dnt" param-type="boolean" required="false" show-location="false" deprecated="false">
  Honor the browser's Do Not Track setting. Default: `false`.
</ParamField>

<ParamField body="opt_out_capturing_by_default" param-type="boolean" required="false" show-location="false" deprecated="false">
  Start with tracking disabled. Requires explicit `opt_in()`. Default: `false`.
</ParamField>

### Consent

<ParamField body="requireConsent" param-type="boolean" required="false" show-location="false" deprecated="false">
  When `true`, no events are sent until `grantConsent()` is called. Default: `false`.
</ParamField>

<ParamField body="autoConsentMode" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Automatically sync consent from Google Consent Mode v2 via the `dataLayer`. When enabled, forces `requireConsent: true`. Pass an object for custom settings: `{ enabled: true, consentParam: 'analytics_storage', dataLayerName: 'dataLayer' }`.
</ParamField>

### Identity

<ParamField body="allowReidentification" param-type="boolean" required="false" show-location="false" deprecated="false">
  Allow calling `identify` with a different user ID after one has already been set. Default: `false` — re-identification is blocked to prevent accidental merges. Call `reset()` before identifying as a different user, or set this to `true` to permit switching identities directly. (An identity previously set only by auto-capture can always be overridden by a later `identify()`.)
</ParamField>

### Session

<ParamField body="session_timeout" param-type="number" required="false" show-location="false" deprecated="false">
  Session timeout in milliseconds. After this period of inactivity, a new session starts. Default: 30 minutes.
</ParamField>

### Debugging

<ParamField body="debug" param-type="boolean" required="false" show-location="false" deprecated="false">
  Enable verbose console logging. Default: `false`.
</ParamField>

<ParamField body="verbose" param-type="boolean" required="false" show-location="false" deprecated="false">
  Even more detailed logging (includes internal state). Default: `false`.
</ParamField>

<ParamField body="test_mode" param-type="boolean" required="false" show-location="false" deprecated="false">
  Log events to console instead of sending them. Useful for development. Default: `false`.
</ParamField>

### HIPAA compliance

<ParamField body="hipaa" param-type="boolean" required="false" show-location="false" deprecated="false">
  Enable HIPAA mode. Applies a `hipaaFilter` to all outgoing events (whitelists safe properties, redacts PHI from URLs, and drops `$set`/`$set_once` person-property mutations) and force-disables autocapture, form identification, input capture, framework extraction, GTM sync, booking/survey listeners, network interception, geolocation, cross-domain tracking, web vitals, error tracking, and cookies. It also forces `respect_dnt`, `mask_all_text`, `mask_all_element_attributes`, `disable_remote_config`, and `persistence: 'localStorage'`. These overrides are immutable — `updateConfig()` cannot re-enable them. Default: `false`.
</ParamField>

<ParamField body="disable_remote_config" param-type="boolean" required="false" show-location="false" deprecated="false">
  Prevent loading server-defined tags, snippets, and transformers from the `/decide` endpoint. Force-enabled in HIPAA mode. Default: `false`.
</ParamField>

<ParamField body="disable_cookie" param-type="boolean" required="false" show-location="false" deprecated="false">
  Disable cookie-based storage entirely. Force-enabled in HIPAA mode. Default: `false`.
</ParamField>

<ParamField body="max_storage_duration" param-type="number" required="false" show-location="false" deprecated="false">
  Maximum hours to retain data in local storage; stale data is purged on init. Default: `24` in HIPAA mode, `0` (disabled) otherwise.
</ParamField>

<Callout kind="alert">
  HIPAA mode reduces the risk of sending sensitive data but does not guarantee compliance. Review your event schemas and legal requirements with your compliance team.
</Callout>

### Masking

<ParamField body="mask_all_text" param-type="boolean" required="false" show-location="false" deprecated="false">
  Mask text content in autocaptured events. Force-enabled in HIPAA mode.
</ParamField>

<ParamField body="mask_all_element_attributes" param-type="boolean" required="false" show-location="false" deprecated="false">
  Mask element attributes in autocaptured events. Force-enabled in HIPAA mode.
</ParamField>

### Callbacks

<ParamField body="loaded" param-type="function" required="false" show-location="false" deprecated="false">
  Called with the Mythic instance after initialization completes. Signature: `(instance) => void`.
</ParamField>

<ParamField body="before_send" param-type="function" required="false" show-location="false" deprecated="false">
  Transform or filter events before sending. Return the event to send it, or `null` to drop it. Signature: `(event) => event | null`.
</ParamField>

<ParamField body="sanitize_properties" param-type="function" required="false" show-location="false" deprecated="false">
  Modify properties before they are attached to an event. Signature: `(properties, eventName) => properties`.
</ParamField>

<ParamField body="on_xhr_error" param-type="function" required="false" show-location="false" deprecated="false">
  Called when a network request fails. Signature: `(error) => void`.
</ParamField>

### Advanced auto-tracking

<ParamField body="auto_form_identify" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically identify users from email fields on form submission. Default: `true`.
</ParamField>

<ParamField body="auto_input_capture" param-type="boolean" required="false" show-location="false" deprecated="false">
  Capture email addresses as they are typed into input fields. Default: `true`.
</ParamField>

<ParamField body="auto_booking_listener" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically detect and track booking API calls (LeadConnector, Calendly, Acuity). Default: `true`. Fires `appointment_booked`, `calendly_booked`, or `acuity_booked`.
</ParamField>

<ParamField body="auto_survey_tracking" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically track survey submission API calls (LeadConnector, Typeform, SurveyMonkey). Default: `true`. Fires `survey_completed`, `typeform_submitted`, or `surveymonkey_completed`.
</ParamField>

<ParamField body="auto_network_intercept" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Intercept fetch/XHR requests to the endpoints listed in `network_endpoints` and capture them as events. Default: `false`.
</ParamField>

<ParamField body="network_endpoints" param-type="array" required="false" show-location="false" deprecated="false">
  Custom endpoints to monitor when `auto_network_intercept` is enabled. Each entry: `{ url, method?, event_name, event_prefix?, extract_fields? }`. `url` supports wildcards (e.g. `*/api/checkout`).
</ParamField>

<ParamField body="auto_framework_extract" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Extract page context from frameworks (Nuxt, Next.js, React, Vue, Angular). Default: `true`. Pass an object for control: `{ enabled: true, framework: 'auto', extract_properties: ['funnelId', 'pageId'] }`.
</ParamField>

<ParamField body="auto_gtm_sync" param-type="boolean" required="false" show-location="false" deprecated="false">
  Sync events to Google Tag Manager's `dataLayer`. Default: `true`.
</ParamField>

<ParamField body="wait_for_page_data" param-type="boolean" required="false" show-location="false" deprecated="false">
  Buffer events until framework extraction completes so the first `$pageview` includes framework-detected properties. Default: `true` when `auto_framework_extract` is enabled. A 2-second safety timeout prevents indefinite blocking; set to `false` to opt out of buffering.
</ParamField>

### Web Vitals

<ParamField body="web_vitals" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Capture Core Web Vitals (LCP, FID, CLS, FCP, TTFB, INP). Default: `false`. Pass an object for fine-grained control: `{ enabled: true, metrics: ['LCP', 'CLS', 'INP'], sample_rate: 0.5 }`.
</ParamField>

### Error tracking

<ParamField body="error_tracking" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Capture uncaught JS errors and unhandled promise rejections as `$exception` events (with type, message, and stack). Default: `true`. Pass an object for control: `{ maxPerPage: 20 }` — a hard cap on exceptions reported per page load so a broken page can't flood ingestion. Grouping into issues happens in the read layer; the SDK ships the raw fields. Force-disabled under HIPAA.
</ParamField>

### Session replay

<ParamField body="session_replay" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Record user sessions (DOM, console, and network activity) with [rrweb](https://github.com/rrweb-io/rrweb) for later playback. Default: `false` (off). Pass `true` to enable with defaults, or an object for control:

  - `enabled` (boolean) — turn recording on. Default: `false`.
  - `sampleRate` (number, 0–1) — fraction of sessions to record, decided deterministically per session ID. Default: `1`.
  - `minDurationMs` (number) — only keep a recording once the session has lasted this long (measured on session age, across page loads). Events buffer from the start, but nothing is transmitted (and no recording appears) until the threshold is crossed, so shorter bounce sessions send zero data. Default: `10000` (10s); set `0` to keep every recording.
  - `maskAllInputs` (boolean) — mask all form input values. Default: `true`.
  - `captureConsole` (boolean) — record `console` log/warn/error calls. Default: `true`.
  - `captureNetwork` (boolean) — record fetch/XHR/resource timing (a waterfall). Default: `true`.
  - `recordNetworkHeaders` (boolean) — include request/response headers. Default: `false`.
  - `recordNetworkBody` (boolean) — include response bodies (capped at 10&nbsp;KB). Default: `false`.
  - `blockSelector` (string) — CSS selector for elements to exclude from the recording.
  - `maskTextSelector` (string) — CSS selector for text nodes to mask.
  - `flushEventThreshold` (number) — buffered events before an upload. Default: `100`.
  - `flushIntervalMs` (number) — maximum milliseconds between uploads. Default: `5000`.
</ParamField>

<Callout kind="info">
  Session replay is normally turned on **per client from the server** (via the `/decide` remote config). The server value wins over your local config: an explicit server `enabled: false` disables recording even if you pass `enabled: true` here, and recording only starts after remote config loads. Recording is blocked entirely for HIPAA locations. Recorded sessions are read back through the [Data API](/data-api/overview) and expire after 30 days.
</Callout>

### Geolocation

<ParamField body="geolocation" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Enrich events with IP-based geolocation data. Default: enabled at city level. Pass an object for control: `{ enabled: true, level: 'country' | 'region' | 'city', cache_duration: 86400000, api_endpoint: '...' }`. `cache_duration` is in milliseconds (default `86400000`, 24 hours). Set `api_endpoint: false` to skip the external lookup API, or set the whole option to `false` to disable geolocation.
</ParamField>

### Cross-domain tracking

<ParamField body="cross_domain_tracking" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Track users across domains by decorating links with session IDs. Opt-in only. Default: `false`. Pass `{ enabled: true, domains: ['app.example.com', 'docs.example.com'] }`.
</ParamField>

### Rate limiting

<ParamField body="rate_limiting" param-type="object" required="false" show-location="false" deprecated="false">
  Client-side rate limiting. Example: `{ events_per_second: 10, events_burst_limit: 50 }`.
</ParamField>

### Properties and forms

<ParamField body="super_properties" param-type="object" required="false" show-location="false" deprecated="false">
  Properties merged into every event from initialization. Equivalent to calling `set()` at startup. Default: `{}`.
</ParamField>

<ParamField body="form_selector" param-type="string" required="false" show-location="false" deprecated="false">
  CSS selector for forms tracked by `auto_form_identify`. Default: `'form, .c-form, [data-form]'`.
</ParamField>

<ParamField body="sensitive_fields" param-type="array" required="false" show-location="false" deprecated="false">
  Field names to strip from captured form/network data (passwords, tokens, card numbers, etc.). Your list is used in place of the built-in defaults.
</ParamField>

<ParamField body="max_payload_size" param-type="number" required="false" show-location="false" deprecated="false">
  Maximum size, in characters, of captured network/form payloads. Default: `6000`.
</ParamField>

<ParamField body="properties_string_max_length" param-type="number" required="false" show-location="false" deprecated="false">
  Maximum length for individual string property values. Default: `65535`.
</ParamField>

<ParamField body="disabled" param-type="boolean" required="false" show-location="false" deprecated="false">
  Completely disable tracking. Default: `false`.
</ParamField>

<ParamField body="disable_external_storage" param-type="boolean" required="false" show-location="false" deprecated="false">
  Disable localStorage and cookies, keeping data in memory only. Default: `false`.
</ParamField>
