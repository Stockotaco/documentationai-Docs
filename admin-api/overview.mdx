## Overview

The Admin API groups the operational surfaces of the Mythic Analytics platform: Cloudflare KV inspection and cleanup, Google Cloud Pub/Sub subscription management, and admin management of destinations, Airbyte connections, and BigQuery exports. All of these are true admin routes protected by the admin key.

## Base URL

All endpoints are served from the platform host at the root path (no version prefix).

```text
https://mythic-analytics.gulp.workers.dev
```

KV management lives under `/admin/kv`, Pub/Sub under `/admin/pubsub`, destinations under `/admin/destinations`, Airbyte under `/admin/airbyte`, and BigQuery exports under `/admin/bigquery-exports`.

## Response shapes

Unlike the versioned `/api/v1` resource APIs, these endpoints return endpoint-specific JSON objects rather than a shared `{ success, data }` envelope. Success payloads vary per endpoint (see each reference). On failure, endpoints return a bare error object:

<ResponseField name="error" field-type="string" required="true" deprecated="false">
  Human-readable error message. Check the HTTP status code alongside it.
</ResponseField>

## Key concepts

| Concept | Description |
|---------|-------------|
| **KV Store** | Cloudflare KV namespaces (`MYTHIC_KEYS`, `MYTHIC_CFG`) holding key mappings and tenant config |
| **Pub/Sub** | Google Cloud Pub/Sub subscriptions on the configured events topic |
| **Subscription** | A named consumer (push or pull) that receives messages from the topic |

## Error handling

| Status | Meaning |
|--------|---------|
| `400` | Invalid namespace, missing required field, or validation error |
| `401` | Admin key missing or invalid |
| `404` | Key not found (KV `get`) |
| `500` | Upstream/GCP error or internal error |

<Callout kind="info">
  Check both the HTTP status code and the `error` field in the response body for programmatic error handling.
</Callout>
