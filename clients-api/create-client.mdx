## Create client: POST /clients

Create a new client tenant. The API auto-generates `publishable_key` and `secret_key` credentials.

### Headers

<ParamField header="Authorization" param-type="string" required="true" deprecated="false">
  Admin bearer token.
</ParamField>

### Body fields

<ParamField body="id" param-type="string" required="true" deprecated="false">
  Unique client identifier. Must match `^[a-zA-Z0-9_-]{1,255}$`.
</ParamField>

<ParamField body="name" param-type="string" required="true" deprecated="false">
  Human-readable client name, 1-255 characters.
</ParamField>

<ParamField body="status" param-type="boolean" required="false" deprecated="false">
  Whether the client is active. Defaults to `true`.
</ParamField>

<ParamField body="domain" param-type="string" required="false" deprecated="false">
  Primary domain for embedded widgets. Must be a valid domain format.
</ParamField>

<ParamField body="email" param-type="string" required="false" deprecated="false">
  Contact email. Must be a valid email format.
</ParamField>

<ParamField body="phone" param-type="string" required="false" deprecated="false">
  Contact phone number.
</ParamField>

<ParamField body="website" param-type="string" required="false" deprecated="false">
  Public website URL.
</ParamField>

<ParamField body="timezone" param-type="string" required="false" deprecated="false">
  IANA timezone identifier, such as `America/New_York`.
</ParamField>

<ParamField body="global_name" param-type="string" required="false" deprecated="false">
  JavaScript identifier used for the SDK global variable. Must be a valid JS identifier (e.g., `acmeRetail`).
</ParamField>

<ParamField body="company_id" param-type="string" required="false" deprecated="false">
  Internal company or reseller identifier for grouping clients.
</ParamField>

<ParamField body="agency_id" param-type="string" required="false" deprecated="false">
  Agency identifier for multi-tenant agency setups.
</ParamField>

<ParamField body="api_metadata" param-type="object" required="false" deprecated="false">
  Arbitrary JSON metadata attached to the client.
</ParamField>

<ParamField body="ghl_location_id" param-type="string" required="false" deprecated="false">
  External GoHighLevel location identifier for integrations.
</ParamField>

<ParamField body="proxy_api_key" param-type="string" required="false" deprecated="false">
  Scoped API key for proxying traffic or third-party integrations.
</ParamField>

### Example

  ```bash
    -H "Authorization: Bearer YOUR_ADMIN_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "id": "acme-retail",
      "name": "Acme Retail",
      "status": true,
      "domain": "acme-retail.com",
      "email": "ops@acme-retail.com",
      "phone": "+1-415-555-0198",
      "website": "https://acme-retail.com",
      "timezone": "America/Los_Angeles",

      "company_id": "cmp_7kH3nPqR",
      "api_metadata": {
        "plan": "enterprise",

      }
    }'
  ```
</Request>

<Callout kind="info">
  The `publishable_key` (starts with `pk_`) is safe for client-side code. The `secret_key` (starts with `sk_`) must stay on your backend.
</Callout>

Returns `409 Conflict` if a client with the same `id` already exists.
