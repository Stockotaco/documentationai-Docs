## Overview

The Mythic Analytics JavaScript SDK captures events, page views, and user identity data from the browser. It manages batching, sessions, attribution, and remote configuration automatically.

Key capabilities:

- Initialize with a publishable key (`pk_` prefix, safe for client-side use)
- Track custom events with `capture` and page views with `pageview`
- Identify users with `identify` and create aliases with `alias`
- Manage super properties that attach to every event
- Control consent, opt-in/opt-out, and HIPAA compliance
- Auto-capture page views, page leaves, UTM parameters, and click IDs
- Remote configuration for tags, snippets, and transformers

<Callout kind="info">
  The SDK uses a publishable key starting with `pk_`. This key is safe to expose in client-side code. Never use admin or secret keys in the browser.
</Callout>

## Install or load the SDK

<Steps>
  <Step title="Load the script or install from npm" icon="download" titleType="p">
    Use a script tag for quick embeds:

    ```html
    <script
      src="https://cdn.mythic-analytics.com/sdk/latest/mythic.min.js"
      crossorigin="anonymous"
    ></script>
    ```

    Or install from npm for bundlers:

    ```bash
    npm install @mythic/analytics
    # or
    yarn add @mythic/analytics
    ```
  </Step>

  <Step title="Import or reference the global" icon="code" titleType="p">
    <CodeGroup tabs="Script tag,ES module,CommonJS">
      ```html
      <script>
        window.mythic = window.Mythic.init("pk_your_publishable_key");
      </script>
      ```

      ```javascript
      import { init } from "@mythic/analytics";
      const mythic = init("pk_your_publishable_key");
      ```

      ```javascript
      const { init } = require("@mythic/analytics");
      const mythic = init("pk_your_publishable_key");
      ```
    </CodeGroup>

    After this step, you have a `mythic` instance you can call methods on.
  </Step>
</Steps>

## Initialize the SDK

### `init(key, config?)`

Create a configured Mythic client instance.

<ParamField body="key" param-type="string" required="true" show-location="false" deprecated="false">
  Publishable project key starting with `pk_`.
</ParamField>

<ParamField body="config" param-type="object" required="false" show-location="false" deprecated="false">
  Optional configuration overrides. See the full config reference below.
</ParamField>

```javascript
import { init } from "@mythic/analytics";

const mythic = init("pk_your_publishable_key", {
  api_host: "https://analytics.yourdomain.com",
  debug: true,
  batch_size: 20,
  autocapture: true,
  capture_pageview: true,
  hipaa: false,
});
```

## Core tracking methods

All examples assume you have created `const mythic = init(...)`.

### `capture(event, properties?)`

Track a custom event with optional properties.

<ParamField body="event" param-type="string" required="true" show-location="false" deprecated="false">
  Event name, such as `Signup started` or `Checkout completed`.
</ParamField>

<ParamField body="properties" param-type="object" required="false" show-location="false" deprecated="false">
  Additional properties. Merged with super properties and auto-captured context.
</ParamField>

```javascript
mythic.capture("Signup started", {
  plan: "pro_monthly",
  referrer: document.referrer,
  experiment_variant: "landing_b",
});
```

### `pageview(properties?)`

Send a page view event. The SDK auto-includes URL, title, referrer, and performance metrics.

<ParamField body="properties" param-type="object" required="false" show-location="false" deprecated="false">
  Optional additional properties.
</ParamField>

```javascript
// On initial load
mythic.pageview({ page: "/pricing", campaign: "spring_promo" });

// In a SPA router hook
router.afterEach((to) => {
  mythic.pageview({ page: to.fullPath });
});
```

<Callout kind="tip">
  By default the SDK auto-captures page views on load and SPA navigation (History API). You only need to call `pageview()` manually if you disable `capture_pageview` or `capture_spa_pageview`.
</Callout>

### `identify(userId, properties?)`

Associate the current session with a known user ID.

<ParamField body="userId" param-type="string" required="true" show-location="false" deprecated="false">
  Stable identifier for the user, e.g., `usr_a1b2c3d4`.
</ParamField>

<ParamField body="properties" param-type="object" required="false" show-location="false" deprecated="false">
  User properties such as email, plan, or role. Sent as a `$identify` event.
</ParamField>

```javascript
mythic.identify("usr_7f3c92d8", {
  email: "jane@acme.com",
  full_name: "Jane Chen",
  plan: "enterprise",
  team_id: "team_b84f21e7",
});
```

### `alias(alias)`

Create an alias linking the current distinct ID to another identifier.

<ParamField body="alias" param-type="string" required="true" show-location="false" deprecated="false">
  The alias to create. Sends a `$create_alias` event.
</ParamField>

```javascript
mythic.alias("user_email@acme.com");
```

### `reset(options?)`

Clear local user identity and session state. Typically called on logout.

<ParamField body="options" param-type="object" required="false" show-location="false" deprecated="false">
  Optional flags. Pass `{ resetInitialAttribution: true }` to also clear first-touch attribution data.
</ParamField>

```javascript
// Standard reset on logout
mythic.reset();

// Full reset including attribution
mythic.reset({ resetInitialAttribution: true });
```

After `reset`, calls to `capture` use a new anonymous distinct ID until you call `identify` again.

### `flush()`

Force-send all queued events immediately. Use before page unload.

```javascript
window.addEventListener("beforeunload", () => {
  mythic.flush();
});
```

## Super properties

Super properties are automatically merged into every event. Use them for values that should be attached to all subsequent events, like a user's subscription plan or A/B test variant.

### `set(properties)`

Set one or more super properties. Overwrites existing values.

```javascript
mythic.set({ plan: "enterprise", team_size: 50 });
```

### `set_once(properties)`

Set super properties only if they do not already exist.

```javascript
mythic.set_once({ first_seen_campaign: "spring_2024" });
```

### `unset(property)`

Remove a single super property.

```javascript
mythic.unset("temporary_flag");
```

### `get_property(property)`

Retrieve a single super property value.

```javascript
const plan = mythic.get_property("plan"); // "enterprise"
```

### `get_super_properties()`

Retrieve all super properties as an object.

```javascript
const allProps = mythic.get_super_properties();
```

### `clear_super_properties()`

Remove all super properties.

```javascript
mythic.clear_super_properties();
```

## Consent and privacy

### `opt_in()` / `opt_out()`

Control whether the SDK sends events.

```javascript
mythic.opt_in();   // Enable tracking
mythic.opt_out();  // Disable tracking (no events sent)
```

### `has_opted_out()`

Check the current opt-out status.

```javascript
if (mythic.has_opted_out()) {
  showConsentBanner();
}
```

### `grantConsent()` / `revokeConsent()`

For consent-gated setups where `requireConsent: true`. No events are sent until `grantConsent()` is called.

```javascript
// User accepts analytics in your cookie banner
function onConsentAccepted() {
  mythic.grantConsent();
}

// User withdraws consent — clears all stored data
function onConsentDeclined() {
  mythic.revokeConsent();
}
```

<Callout kind="info">
  When `requireConsent` is `true`, the SDK initializes but does not track any events until `grantConsent()` is called. `revokeConsent()` stops tracking and clears all local storage.
</Callout>

## Utility methods

### `getDistinctId()`

Returns the current distinct ID (anonymous or identified).

```javascript
const distinctId = mythic.getDistinctId();
```

### `getSessionId()`

Returns the current session ID.

```javascript
const sessionId = mythic.getSessionId();
```

### `getDeviceId()`

Returns the persistent device ID (5-year expiration).

```javascript
const deviceId = mythic.getDeviceId();
```

### `isBot()`

Returns `true` if the current user agent is detected as a bot.

```javascript
if (mythic.isBot()) return; // Skip tracking for bots
```

### `debug(enabled?)`

Toggle debug logging. When enabled, the SDK logs queued events, network requests, and configuration decisions to the console.

```javascript
mythic.debug(true);  // Enable verbose logging
mythic.debug(false); // Disable
```

### `getConfig()`

Returns the current SDK configuration object.

```javascript
const config = mythic.getConfig();
console.log(config.api_host);
```

### `updateConfig(newConfig)`

Dynamically update configuration options at runtime.

```javascript
mythic.updateConfig({ debug: true, batch_size: 5 });
```

### `refreshConfig()`

Re-fetch remote configuration from the server.

```javascript
await mythic.refreshConfig();
```

## Configuration reference

The full set of options you can pass to `init(key, config)`.

### Network and batching

<ParamField body="api_host" param-type="string" required="false" show-location="false" deprecated="false">
  Base URL for API calls. Default: `https://mythic-analytics.gulp.workers.dev`. Change this if you use a custom domain or proxy.
</ParamField>

<ParamField body="batch_size" param-type="number" required="false" show-location="false" deprecated="false">
  Number of events to queue before sending. Default: `10`.
</ParamField>

<ParamField body="flush_interval" param-type="number" required="false" show-location="false" deprecated="false">
  Milliseconds between automatic flushes. Default: `2000`.
</ParamField>

<ParamField body="xhr_headers" param-type="object" required="false" show-location="false" deprecated="false">
  Custom headers to include with every network request.
</ParamField>

### Auto-capture

<ParamField body="autocapture" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Enable automatic capture of clicks, form submissions, and DOM interactions. Default: `true`. Pass an object to limit to specific element types: `{ elements: ['links', 'buttons', 'forms'] }`.
</ParamField>

<ParamField body="capture_pageview" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture a `$pageview` event on page load. Default: `true`.
</ParamField>

<ParamField body="capture_spa_pageview" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture page views on SPA navigation (History API pushState/replaceState). Default: `true`.
</ParamField>

<ParamField body="capture_pageleave" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture a `$pageleave` event with time-on-page and exit intent data. Default: `true`.
</ParamField>

<ParamField body="capture_utm" param-type="boolean" required="false" show-location="false" deprecated="false">
  Auto-capture UTM parameters and advertising click IDs from the URL. Default: `true`.
</ParamField>

<ParamField body="capture_copied_text" param-type="boolean" required="false" show-location="false" deprecated="false">
  Track text copied to clipboard. Default: `false`.
</ParamField>

### Privacy and persistence

<ParamField body="persistence" param-type="string" required="false" show-location="false" deprecated="false">
  Storage strategy. Options: `'localStorage'`, `'cookie'`, `'memory'`, `'localStorage+cookie'`. Default: `'localStorage+cookie'`.
</ParamField>

<ParamField body="persistence_name" param-type="string" required="false" show-location="false" deprecated="false">
  Custom prefix for storage keys. Defaults to `{globalName}_{key}_`.
</ParamField>

<ParamField body="cookie_name" param-type="string" required="false" show-location="false" deprecated="false">
  Custom cookie name when using cookie persistence.
</ParamField>

<ParamField body="cookie_expiration" param-type="number" required="false" show-location="false" deprecated="false">
  Cookie lifetime in days. Default: `365`.
</ParamField>

<ParamField body="cross_subdomain_cookie" param-type="boolean" required="false" show-location="false" deprecated="false">
  Share cookies across subdomains. Default: `true`.
</ParamField>

<ParamField body="secure_cookie" param-type="boolean" required="false" show-location="false" deprecated="false">
  Set the `Secure` flag on cookies. Default: `true` on HTTPS.
</ParamField>

<ParamField body="respect_dnt" param-type="boolean" required="false" show-location="false" deprecated="false">
  Honor the browser's Do Not Track setting. Default: `false`.
</ParamField>

<ParamField body="opt_out_capturing_by_default" param-type="boolean" required="false" show-location="false" deprecated="false">
  Start with tracking disabled. Requires explicit `opt_in()`. Default: `false`.
</ParamField>

### Consent

<ParamField body="requireConsent" param-type="boolean" required="false" show-location="false" deprecated="false">
  When `true`, no events are sent until `grantConsent()` is called. Default: `false`.
</ParamField>

<ParamField body="autoConsentMode" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Automatically sync consent from Google Consent Mode v2 via the `dataLayer`. When enabled, forces `requireConsent: true`. Pass an object for custom settings: `{ enabled: true, consentParam: 'analytics_storage', dataLayerName: 'dataLayer' }`.
</ParamField>

### Identity

<ParamField body="allowReidentification" param-type="boolean" required="false" show-location="false" deprecated="false">
  Allow calling `identify` with a different user ID on the same device. Default: `true`. Set to `false` to preserve the first identity and prevent accidental merges.
</ParamField>

### Session

<ParamField body="session_timeout" param-type="number" required="false" show-location="false" deprecated="false">
  Session timeout in milliseconds. After this period of inactivity, a new session starts. Default: 30 minutes.
</ParamField>

### Debugging

<ParamField body="debug" param-type="boolean" required="false" show-location="false" deprecated="false">
  Enable verbose console logging. Default: `false`.
</ParamField>

<ParamField body="verbose" param-type="boolean" required="false" show-location="false" deprecated="false">
  Even more detailed logging (includes internal state). Default: `false`.
</ParamField>

<ParamField body="test_mode" param-type="boolean" required="false" show-location="false" deprecated="false">
  Log events to console instead of sending them. Useful for development. Default: `false`.
</ParamField>

### HIPAA compliance

<ParamField body="hipaa" param-type="boolean" required="false" show-location="false" deprecated="false">
  Enable HIPAA mode. Applies a `hipaaFilter` to all outgoing events and force-disables autocapture, form identification, geolocation, cross-domain tracking, cookies, AI chat, and other features that could expose PHI. Default: `false`.
</ParamField>

<ParamField body="max_storage_duration" param-type="number" required="false" show-location="false" deprecated="false">
  Maximum hours to retain data in local storage. Only relevant in HIPAA mode. Default: `24` (in HIPAA mode).
</ParamField>

<Callout kind="alert">
  HIPAA mode reduces the risk of sending sensitive data but does not guarantee compliance. Review your event schemas and legal requirements with your compliance team.
</Callout>

### Masking

<ParamField body="mask_all_text" param-type="boolean" required="false" show-location="false" deprecated="false">
  Mask text content in autocaptured events. Force-enabled in HIPAA mode.
</ParamField>

<ParamField body="mask_all_element_attributes" param-type="boolean" required="false" show-location="false" deprecated="false">
  Mask element attributes in autocaptured events. Force-enabled in HIPAA mode.
</ParamField>

### Callbacks

<ParamField body="loaded" param-type="function" required="false" show-location="false" deprecated="false">
  Called with the Mythic instance after initialization completes. Signature: `(instance) => void`.
</ParamField>

<ParamField body="before_send" param-type="function" required="false" show-location="false" deprecated="false">
  Transform or filter events before sending. Return the event to send it, or `null` to drop it. Signature: `(event) => event | null`.
</ParamField>

<ParamField body="sanitize_properties" param-type="function" required="false" show-location="false" deprecated="false">
  Modify properties before they are attached to an event. Signature: `(properties, eventName) => properties`.
</ParamField>

<ParamField body="on_xhr_error" param-type="function" required="false" show-location="false" deprecated="false">
  Called when a network request fails. Signature: `(error) => void`.
</ParamField>

### Advanced auto-tracking

<ParamField body="auto_form_identify" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically identify users from email fields on form submission. Default: `true`.
</ParamField>

<ParamField body="auto_input_capture" param-type="boolean" required="false" show-location="false" deprecated="false">
  Capture email addresses as they are typed into input fields. Default: `true`.
</ParamField>

<ParamField body="auto_booking_listener" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically detect and track booking widget interactions (Calendly, etc.). Default: `false`.
</ParamField>

<ParamField body="auto_survey_tracking" param-type="boolean" required="false" show-location="false" deprecated="false">
  Automatically track survey widget responses. Default: `false`.
</ParamField>

<ParamField body="auto_network_intercept" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Intercept fetch/XHR requests to specified endpoints and capture them as events. Default: `false`.
</ParamField>

<ParamField body="auto_framework_extract" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Extract page metadata from frameworks (Nuxt, Next.js, React, Vue, Angular). Default: `false`. Pass `{ framework: 'auto' }` for auto-detection.
</ParamField>

<ParamField body="auto_gtm_sync" param-type="boolean" required="false" show-location="false" deprecated="false">
  Sync events to Google Tag Manager's `dataLayer`. Default: `true`.
</ParamField>

<ParamField body="wait_for_page_data" param-type="boolean" required="false" show-location="false" deprecated="false">

### Web Vitals

<ParamField body="web_vitals" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Capture Core Web Vitals (LCP, FID, CLS, FCP, TTFB, INP). Default: `false`. Pass an object for fine-grained control: `{ enabled: true, metrics: ['LCP', 'CLS', 'INP'], sample_rate: 0.5 }`.
</ParamField>

### Geolocation

<ParamField body="geolocation" param-type="boolean|object" required="false" show-location="false" deprecated="false">
  Enrich events with geolocation data. Default: enabled at city level. Pass an object for control: `{ enabled: true, level: 'country' | 'region' | 'city', cache_duration: 3600 }`. Set to `false` to disable.

### Cross-domain tracking

<ParamField body="cross_domain_tracking" param-type="boolean|object" required="false" show-location="false" deprecated="false">

</ParamField>

### Rate limiting

<ParamField body="rate_limiting" param-type="object" required="false" show-location="false" deprecated="false">
  Client-side rate limiting. Example: `{ events_per_second: 10, events_burst_limit: 50 }`.
</ParamField>

## Auto-captured events

The SDK automatically captures these events when the corresponding config options are enabled:

| Event | Description | Config |
|-------|-------------|--------|
| `$pageview` | Page load with URL, title, referrer, and performance metrics | `capture_pageview` |
| `$pageleave` | Page exit with `$time_on_page`, `$exit_intent`, scroll depth | `capture_pageleave` |
| `$engagement` | Fires 10 seconds after page load | Always on |
| `$identify` | User identification (via `identify()`) | Always on |
| `$create_alias` | Alias creation (via `alias()`) | Always on |
| Click events | Auto-captured link clicks, button clicks | `autocapture` |
| Form submissions | Auto-captured form submit events | `autocapture` |

### Attribution tracking

When `capture_utm` is enabled (default), the SDK captures UTM parameters and advertising click IDs from the URL:

- **UTM**: `$utm_source`, `$utm_medium`, `$utm_campaign`, `$utm_term`, `$utm_content`
- **Google Ads**: `$gclid`, `$gbraid`, `$wbraid`
- **Facebook**: `$fbclid`, `$fbc`, `$fbp`
- **Twitter/X**: `$twclid`
- **LinkedIn**: `$li_fat_id`
- **Mythic Click ID**: `$iiid` (from link redirect tracking)

### Auto-included context

Every event automatically includes:

- `$lib`, `$lib_version` — SDK identifier and version
- `$current_url`, `$referrer`, `$host`, `$pathname` — Page context
- `$browser`, `$browser_version`, `$os`, `$os_version` — User agent
- `$device_type` — `mobile`, `tablet`, or `desktop`
- `$screen_width`, `$screen_height`, `$viewport_width`, `$viewport_height`
- `$session_id`, `$device_id`, `distinct_id` — Identity context

## Troubleshooting

### No events appearing

<Steps>
  <Step title="Verify initialization" icon="check-circle" titleType="p">
    Confirm `init` is called with a valid publishable key:

    ```javascript
    const mythic = init("pk_your_publishable_key");
    ```
  </Step>

  <Step title="Enable debug logging" icon="terminal" titleType="p">
    ```javascript
    const mythic = init("pk_your_publishable_key", { debug: true });
    ```

    You should see logs when `capture` or `pageview` fires and when events are sent.
  </Step>

  <Step title="Check network requests" icon="globe" titleType="p">
    Open the browser network panel and filter by `/e?key=` or `/decide?key=`. Verify requests reach the expected `api_host`.
  </Step>
</Steps>

### Network requests blocked

- If blocked by ad blockers or CSP, serve Mythic via a custom subdomain and update `api_host`.
- If requests fail with `4xx`, check that the key is correct and not expired.
- If requests fail with `5xx`, retry later or contact support.

### Wrong host or key

```javascript
// Fix the configuration and redeploy
const mythic = init("pk_your_correct_key", {
  api_host: "https://analytics.yourdomain.com",
});
```

Verify in the network panel that the `key` query parameter matches your Mythic project key.
